1. Mise en garde

Comme pour toute installation d'application dynamique, il est important de garder celle-ci à jour. Si vous ne désirez pas mettre à jour votre site vous risquez de l'exposer à diverses vulnérabilités causées par des failles de sécurité. Il est important de rappeler que le fait d'utiliser n'importe quelle application dynamique représente un risque, celle-ci pourrait encore contenir une ou plusieurs failles. Afin de mieux protéger votre site, vous pouvez vous tenir informé des dernières annonces en vous inscrivant à la mailing-list (en anglais).

2. Recommandations

Système d'Exploitation
Toute application nécessite un système d'exploitation pour fonctionner. Les types de serveurs les plus connus sont : Microsoft Windows, UNIX and Linux ainsi que Apple MacOS. La vaste majorité des serveurs internet utilisent Linux, il y a donc plus de documentation et de support fournit pour les utilisateurs de Linux. Zikula s'efforce d'être indépendant du système d'exploitation et il est dans ce but intensivement testé sous Linux et Windows.
Serveur internet
Les deux serveurs internet les plus connus sont Apache et Microsoft IIS. Apache est le serveur internet préféré des internautes et la plupart des références à un serveur internet concernent Apache. Zikula est indépendant du serveur internet, bien que la majorité des tests soit réalisé avec Apache.
PHP
PHP(Hypertext Pre-processor) est un script qui est exécuté du côté serveur. Zikula requiert, actuellement, de disposer au minimum de la version 4.3 de PHP bien que celle-ci ne soit plus supportée par ses créateurs depuis janvier 2008. Il est donc recommandé d'utiliser au moins la dernière version stable de PHP.
Base de données
Zikula a aussi besoin d'une base de données pour pouvoir être installé et supporte actuellement MySQL 5 au minimum, Microsoft SQL Server (MSSQL), PostgreSQL, and Oracle. Cette version de Zikula est la première qui étend son support à autre chose que MySQL. Le support des bases de données autre que MySQL est assuré par l'API et donc les modules additionnels qui n'utilisent pas l'API de Zikula peuvent ne pas être compatible.
Zikula
Chaque version a une durée de vie variable qui dépend de la date de sortie de la prochaine version stable. Nous vous recommandons d'utiliser la dernière version stable aussi vite que possible après que cette dernière aie été publiée et ce pour des raisons de sécurité, de performance et de support.
Modules Additionnels
Ce sont eux qui fournissent à Zikula ses vrais fonctionnalités. Veillez à utiliser la dernière version stable de ceux-ci également.

3. Informations concernant Apache

Apache est le meilleur choix à faire pour un serveur internet. Il est recommandé que vous utilisiez la dernière version stable de Apache, mais ceci n'est pas obligatoire. Si vous utilisez Apache, veuillez installer ces extensions si il vous est possible de le faire :

mod_security
mod_security est un script open source qui permet de protéger les applications internet.
mod_rewrite
mod_rewrite permet d'utiliser l'url rewriting(modifier l'URL).

4. Fichiers de configuration du serveur

Sous Apache, ils sont représentés par des fichiers .htaccess. Ils vous permettent de modifier la configuration de votre serveur pour le répertoire dans lequel il est placé. Cela permet par exemple de modifier la configuration de PHP(si PHP est exécuté comme étant un module de Apache) ou encore de rendre l'utilisation de l'url rewriting possible et bien d'autres choses encore. Il est important de rappeler qu'une mauvaise utilisation de ces fichiers peut entrainer une erreur du serveur internet. Ils ne sont pas requis par Zikula mais Zikula et beaucoup de modules additionnels essayent d'en tirer avantage afin d'améliorer la sécurité et les fonctionnalités.

Sur certains serveurs d'hébergement il est possible d'ajouter un fichier php.ini dans vos répertoires afin de modifier la configuration de PHP.

5. Configuration de PHP

Bien que Zikula ne requiert pas une installation exotique de PHP, une bonne configuration de PHP est importante. Si vous avez la possibilité de modifier la configuration de PHP, vous devriez prendre ceci en compte :

register_globals
Register_globals devrait être sur off pour des raisons de sécurité. Note : le fichier .htaccess par défaut met cette option sur off grâce à un flag.
magic_quotes_gpc
Magic_quotes_gpc devrait être sur off pour des raisons de sécurité. Note : le fichier .htaccess par défaut met cette option sur off grâce à un flag.
magic_quotes_runtime
magic_quotes_runtime doit être sur off.
gd support
La librairie GD est utilisée pour manipuler des images grâce au serveur internet et par des galeries d'images.
allow_url_fopen
allow_url_fopen de préférence sur off pour des raisons de sécurité.
allow_url_include
allow_url_include de préférence sur off.
display_errors
display_errors devrait être normalement sur off, il existe une option pour le mettre sur on dans le fichier /config/config.php en cas de besoin.
display_startup_errors
display_startup_errors devrait être sur off.
expose_php
