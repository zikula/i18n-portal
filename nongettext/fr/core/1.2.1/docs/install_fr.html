1. Zikula : Version Forest

Zikula est un framework php d'application internet open source pour des sites internet dynamiques. Le coeur du système inclut une API complète à partir de laquelle des développements tiers peuvent être créés. Zikula dipose d'une large communauté qui fournit un grand nombre d'extensions au système et un support via ses forums.

Nous acceptons les contributions de tout un chacun, peu importe le niveau de compétence. Vous pouvez nous aider.

2. Recommandations

Avant de commencer, il existe certaines recommandations et indications que vous devriez prendre en compte afin d'installer, de faire fonctionner et de sécuriser Zikula facilement. Veuillez lire le document concernant les pré-requis.

3. Nouvelle Installation

Il existe deux façon d'installer Zikula :

  • Installation via une interface graphique
  • Installation via un dump SQL

Installation via une interface graphique

Pour exécuter l'installation, ouvrez à l'aide de votre navigateur internet le fichier install.php qui est situé dans le répertoire de Zikula, ex. http://www.exemple.com/install.php. Si vous avez installé Zikula dans un sous-répertoire 'foo' l'URL sera http://www.exemple.com/foo/install.php.

Des informations sont fournies durant toute l'installation. Zikula requiert qu'une base de données soit crée à une étape de l'installation, le fait de créer cette base de données avant d'exécuter l'installation permet d'éviter d'avoir des problèmes par la suite. Bien qu'il soit possible de créer une base de données durant l'installation, cela ne fonctionne pas toujours car il est nécessaire d'avoir un droit d'accès élevé aux bases de données. Chez la plupart des hébergeurs ce n'est pas le cas, il est donc recommandé de créer cette base de données avant via votre système d'administration. Si vous faites cela, notez bien le nom de la base de données ainsi que l'identifiant et le mot de passe qui permettent d'y accéder car ils vous seront demandés durant l'installation.

Lorsque la procédure d'installation est terminée, vous pouvez vous rendre sur votre site Zikula en affichant la page index.php dans votre navigateur internet, ex. http://www.exemple.com/index.php. Dans la plupart de cas, cette page sera ouverte automatiquement lorsque vous ouvrirez l'URL http://www.exemple.com.

Installation via un dump SQL (Non recommandé)

Ce type d'installation n'est pas recommandé pour la plupart des cas !
Si vous désirez important un dump SQL pour installer Zikula manuellement, vous pouvez le faire grâce à un script comme phpMyAdmin. Le dump fournit provient d'une base de données sous MySQL 5.x et ne fonctionne pas sous un autre type de base de données ou une autre version de MySQL. Lorsque vous aurez importé le dump SQL, il vous faudra éditer le fichier config/config.php afin d'y ajouter les informations appropriées concernant votre base de données et ce dans la zone réservée à la connexion 'default'. Le nom d'utilisateur et le mot de passe devraient être encodé en base64 pour des raisons de sécurité (il n'est pas difficile de trouver un encodeur sur internet). Le nom d'utilisateur et le mot de passe du compte administrateur pour cette installation sont : admin/Password.

4. Note finale

Les modifications apportées au core ou à la base de données ne sont pas supportées. Le faire peut entrainer d'importants problèmes lors de mises à jour, c'est pour cette raison que ces "hacks" ne sont pas recommandés. Zikula dispose d'un système d'extensions flexible et d'un système de configuration permettant de tout personnaliser, nous vous recommandons de consulter la documentation pour les développeurs à ce propos.

Nous espérons que vous serez heureux d'utiliser cette version de Zikula.
