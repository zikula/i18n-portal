Instructions de Mise à jour pour Zikula

Une documentation plus détaillée concernant l'installation est disponible dans le wiki sur http://communaute.zikula.fr. Un support et de l'aide concernant l'installation et la mise à jour vers Zikula est fournie sur les forums. Il s'agit juste de quelques informations qui doivent s'appliquer à la majorité des hébergeurs.

  • 2. Mise à jour vers Zikula 1.2.2
  • 3. Attention

1. Pré-requis

Zikula peut être mis à jour vers la version 1.2.2 à partir des versions suivantes : 1.1.0, 1.1.1 or 1.1.2. Si votre installation est antérieur à ces versions, veuillez d'abord réaliser la mise à jour vers l'une de ces versions avant de continuer à suivre les insctructions.

Avant de commencer la mise à jour, il existe certaines recommandations et indications que vous devriez prendre en compte afin d'installer, de faire fonctionner et de sécuriser Zikula facilement. Veuillez lire le document concernant les pré-requis.

2. Mise à jour vers Zikula 1.2.2

Respectez l'ordre des étapes indiquées ci-dessous et lisez chacune d'entre elles en détail avant de commencer. Si vous suivez à la lettre les instructions, la mise à jour devrait se faire sans aucun problème.

  • 2.1 Sauvegarde de votre base de données et de vos fichiers
  • 2.2 Préparer votre version existante
  • 2.3 Exécuter la mise à jour
  • 2.4 Nettoyage et configuration additionnelle

2.1 Sauvegarde de votre base de données et de vos fichiers

Il est vital de sauvegarder votre base de données et vos fichiers avant d'exécuter la mise à jour. Il n'est pas possible de revenir en arrière après une mise à jour, dans ce cas la seule solution pour récupérer les informations en cas de problème est de restaurer les données sauvegardées. Pour sauvegarder votre base de données, essayez en utilisant un outil tel que mysqldumper, phpMyAdmin ou via SSH ou encore votre le système d'administration de votre hébergement. Des instructions complètes et des tutoriels à propos de la sauvegarde d'une base de données peuvent être trouvés en cherchant sur le web ou dans la documentation de l'outil que vous désirez utiliser.

--->Vous aurez besoin de la version original du fichier config/config.php par la suite donc ne l'écrasez pas, ne le supprimez pas et mettez le en lieu sur.<---

2.2 Préparer votre version existante

  1. Connectez-vous à votre site Zikula avec un compte administrateur. Si vous le désirez, vous pouvez mettre votre site hors ligne durant la mise à jour en modifiant la configuration dans index.php?module=Settings&type=admin.
  2. Déplacez le module system/legal dans modules/legal. Si vous n'avez pas personnalisé le module legal vous pouvez alors tout simplement le supprimer du dossier system/.
  3. Si vous avez le module Profile dans le dossier system/, déplacez system/Profile dans modules/Profile. Si vous n'avez pas personnalisé le module Profile vous pouvez alors tout simplement le supprimer du dossier system/. Si le module Profile est déjà dans le dossier modules/ alors ignorez cette étape.
  4. Envoyez tous les fichiers contenu dans le pack que vous avez téléchargé, en écrasant donc les fichiers existants.
  5. Restaurez le fichier config.php à partir de votre sauvegarde que vous avez effectuée à l'étape 2.1.

2.3 Exécuter la mise à jour

Exécutez le script upgrade.php et suivez les étapes indiquées. Il n'y a aucune vrai interaction en dehors du fait de cliquer sur 'suivant'. Tous les changements nécessaires dans le base de données seront fait automatiquement. A la fin, vous serez redirigé vers le panneau d'administration.

2.4 Nettoyage et configuration additionnelle

Après une mise à jour réussie vous pouvez supprimer

  1. Le répertoire install et les fichiers install.php, upgrade.php du répertoire principal de votre site.
  2. Si vous utilisez vos propre thèmes, sachez que désormais les thèmes requierts des informations spécifiques dans leurs header qui doivent être mises à jour pour prendre en compte les balises des plugins lang, langdirection et charset. Veuillez vous référer au template master.htm de andreas08 et voodoodolly, celui-ci comprend les nouvelles modifications au format XHTML 1.0 STRICT et HTML 1.1.

Si vous avez mis votre site hors ligne avant de faire la mise à jour (voir 2.2) vous devrez le remettre en ligne.

3. Attention

Il est important de noter qui Zikula ne supporte aucun changement dans le code du coeur ou la structure des données. Zikula v1.2.2 a été pensé de façon assez flexible afin d'éviter tout modification du coeur, mais si vous remarquez qu'il manque quelque chose que vous considérez comme important, veuillez en faire la demande sur code.zikula.org/core.

4. Note finale
